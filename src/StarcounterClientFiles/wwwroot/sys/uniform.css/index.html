Lightweight, opinionated CSS library that plays nicely within Shadow DOM.

Underwear.css

Usually, it's recommended to put some Underwear before putting the Uniform.

Underwear is a minimalistic library that normalizes User Agent styles and puts a tiny layer of sugar on top. They are designed to play well together.

  • Underwear unifies your light DOM content, using just native HTML semantics,
  • Uniform allows you to put your opinionated styling, with specific classes into shadow roots.

    • You can try how it looks with or without Underwear.

    Put Underwear.css for this document (light DOM).
    Put Underwear.css for all sample shadow roots.

    Please note that above distintion works best for browsers that support Shadow DOM, otherwise styles may leak. Without Shadow DOM, outer underwear may bleed into shadow roots, and inner underwear may leak outside. Uniform selectors are more specific, explicit and should not make so many problems.

Custom Properties

Uniform.css uses CSS Custom Properties (CSS Variables) that you can configure per (shadow) root in which you included uniform.css. You can also use them by yourself, to adhere to the consistent look and feel.

<style>.pinky-host{ --uni-primary: pink;}</style>
<div class="pinky-host">
    <shadowroot mode="open">
        <style>
            @import 'uniform.css';
            .app-header{
              color: var(--uni-danger);
            }
        </style>
        <h3 class="app-header"><slot name="app/some"></slot></h3>
        <div class="uni-primary-button"><slot name="app/button"></slot></div>
    </shadowroot>
    <custom-element slot="app/content">Some content with consistent uni-colors</custom-element>
    <button slot="app/button">Pinky primary button</button>
</div>
Some content with consistent uni-colors

Form control

.uni-label-control

Puts slotted input in the left column, everything else in the right column.

It works for distributed elements.

<shadowroot mode="open">
    <style>@import 'uniform.css';</style>
    <div class="uni-label-control">
        <slot></slot>
    </div>
</shadowroot>
<label>Label</label>
<input>

You can use it as well for styling any elements in the same tree.

<shadowroot mode="open">
    <style>@import 'uniform.css';</style>
    <div class="uni-label-control">
        <label>Label</label>
        <slot></slot>
    </div>
</shadowroot>

Buttons

.uni-*-button

Makes a specific type of a button

We would love to use just .uni-*-button class, but we have to use suffix and demand an additional wrapping element to support polyfilled browsers. You can read more at README.md

You can style distributed buttons.

<shadowroot mode="open">
    <style>@import 'uniform.css';</style>
    <div class="uni-success-button">
        <slot></slot>
    </div>
</shadowroot>
<button>light</button>

As well as the elements from the same tree

<shadowroot mode="open">
    <style>@import 'underwear.css';</style>
    <style>@import 'uniform.css';</style>
    <div class="uni-success-button">
        <slot></slot>
        <button>shadow</button>
    </div>
</shadowroot>
<button>light</button>

Pagination

.uni-pagination, .uni-previous, .uni-next, .uni-pages

To style pagination widget use:

  • .uni-pagination - to wrap all page-related links,
  • .uni-previous - to wrap the "previous", or "first" link,
  • .uni-next - to a list of pages,
  • .uni-pages - to wrap the "next", or "last" link,
  • [disabled] - to disable a link, simply put "disabled" attribute,
  • [aria-current="true"] - to mark current page, use ARIA attribute.

<shadowroot mode="open">
    <style>
        @import "uniform.css";
    </style>
    <div class="uni-pagination">
        <div class="uni-previous"><slot name="foo/previous"></slot></div>
        <div class="uni-pages"><slot></slot></div>
        <div class="uni-next"><slot name="foo/next"></slot></div>
    </div>
</shadowroot>
<span slot="foo/previous" onclick="doYourJobInJS()" href="orUseStaticLinks">Previous</span>
<span>2</span>
<span aria-current="true">3</span>
<span>4</span>
<span disabled>5</span>
<span slot="foo/next" disabled>Next</span>
Previous 2 3 4 5 Next

As with other Uniform classes you can use them to style elements from the same tree as you style distributed elements.

Sections

To create different kinds of sections, use:
  • .uni-section-primary - to have a greyish background,
  • .uni-section-secondary - to have white background,
  • .uni-section-tertiary - to have dark blue background
Note, that those sections stick to the sides of padded host. 
<shadowroot mode="open">
    <style>@import 'uniform.css';</style>
    <div class="uni-section-primary">
        Primary section
    </div>
    <section class="uni-section-secondary">
        Secondary section
    </section>
    <div class="uni-section-tertiary">
        Tertiary section
    </div>
</shadowroot>
You can nest cards within sections. It is recommended to lay out cards within sections using CSS Grid Layout. 
<shadowroot mode="open">
    <style>@import '../underwear.css/underwear.css';</style>
    <style>@import 'uniform.css';</style>
    <div class="uni-section-primary">
        <div class="uni-card">Default card in a section.</div>
    </div>
    <div class="uni-section-secondary">
        <div class="uni-card" style="display: block">Full-width card in a section, achieved by setting <code>display: block</code> on the card</div>
    </div>
    <div class="uni-section-primary" style="display: grid;">
        <div class="uni-card">Full-width card in a section, achieved by setting <code>display: grid</code> on the section.</div>
    </div>
    <div class="uni-section-tertiary" style="display: grid; grid-template-columns: repeat(2, 1fr);grid-gap: var(--uni-section-padding-horizontal)">
        <div class="uni-card">Two half-width cards in a section, achieved by setting <code>display: grid</code> on the section (with two columns and a gap).</div>
        <div class="uni-card">Two half-width cards in a section, achieved by setting <code>display: grid</code> on the section (with two columns and a gap).</div>
    </div>
</shadowroot>
